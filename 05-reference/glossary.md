# Glossary

## Goal

Define canonical terminology for Foundry Flow so all specification sections use one consistent vocabulary.

## Canonical Runtime Terms

Provide normative definitions for core runtime terms used across concepts, flow, node, and SDK documents.

## Governance and Legal Terms

Define jurisdiction, tier, hearing, ruling, and finality terms used by governance mechanisms.

## Capability and Contract Terms

Define capability syntax and contract vocabulary (`entryContracts`, `exitContracts`, `entry`, `exit`, `importNode`).

## Data and Provenance Terms

Define Workitem, artefact, version, feedback, stamp, passport, and provenance semantics.

## Superseded Terms

List legacy terms that are explicitly out of scope in v1 and provide the replacement term where applicable.

## Cross-Reference Index

Link each term to its first normative detail location.

## Glossary Invariants

Capture terminology rules that must remain stable across the full spec.
