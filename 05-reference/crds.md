# CRD Reference

## Goal

Provide canonical CRD surfaces and field semantics for all runtime resources.

## Resource Inventory

List the CRDs in scope and their runtime ownership boundaries.

## FoundryFlow

Define flow-level schema, including topology, contracts, policy limits, and cross-flow settings.

## FoundryNode

Define node-level schema, including outputs, capability grants, timeout budget, and entry/exit bindings.

## Workitem

Define declaration and runtime surfaces, lifecycle fields, assignment fields, and artefact references.

## GovernedArtefact

Define artefact-kind governance schema, required stamps, and contract-facing behaviour.

## Law Object Surfaces

Define law schema with goal, representations, tier, and lifecycle metadata.

## Contract and Binding Shapes

Define `entryContracts`, `exitContracts`, `entry`, `exit`, and `importNode` shapes and validation rules.

## Validation and Rejection Rules

List schema and semantic constraints that cause admission rejection.

## CRD Invariants

Capture non-negotiable field and ownership rules that must remain stable.
